# Rejected Codex Suggestions — Round 2

## R2-AR-001 (MAJOR, scope_creep) — REJECTED
**Codex argument:** The "Skill canonical paths" anchor row hard-codes four workflow skills and user-local ~/.claude paths. The bounded N1+N2+N4 task only requires changes around app/core/dependencies.py and app/services/whale_orchestrator.py, so this adds execution requirements outside the requested fix scope. Fix proposed: remove the binding skill/path requirement.
**Motivation for rejection:** Required-by-spec. /tracer-feature command Step 0.2 (~/.claude/commands/tracer-feature.md) EXPLICITLY mandates that the anchors table contain "Skill canonical paths" as one of the immutable rows. An anchor row required by the workflow protocol is not scope creep — it is the contract /tracer-feature consumes downstream. Removing the row would break Step 0.2 invariants and prevent /tracer-feature from operating against the anchors. The row stays. The hard rule (rule 4) of this Round 3 task explicitly designates AR-001 as INVALID for this reason.

## R2-AR-002 (MINOR, scope_creep) — REJECTED
**Codex argument:** The "Command paths" anchor row anchors /tracer-feature, /plan-hardened, /refactor, and /fix-until-green orchestration choices. Those are workflow decisions not required by the N1, N2, or N4 facts and can turn a bounded implementation task into a tool-routing task. Fix proposed: restrict or remove command routing from immutable anchors.
**Motivation for rejection:** Required-by-spec. /tracer-feature command Step 0.2 EXPLICITLY mandates "Command paths" as one of the immutable anchor rows. The four commands listed are the only orchestration surface the tracer interacts with: /tracer-feature drives, /plan-hardened gates, /refactor is the negative-discriminator (used to confirm tracer is correct tool), /fix-until-green is the recovery loop. Pruning these would leave the tracer without an anchored answer to "which command runs which phase". The row stays. Rule 5 of this Round 3 task explicitly designates AR-002 as INVALID for this reason.
